# Security Policy

If you find a security issue that could affect users, classrooms, or distributed binaries, please do not open a public issue first.

## Private Reporting

Preferred order:

1. Use GitHub private vulnerability reporting if it is enabled for this repository
2. Contact the maintainer through the contact information listed on the maintainer's GitHub profile

## What to Include

- summary of the issue
- impact scope
- reproduction steps
- affected platform and version
- logs, screenshots, or a proof of concept
- your estimate of the severity

## Our Target Response

- acknowledge receipt within `7` days when possible
- reproduce and assess impact before deciding on priority
- coordinate disclosure after a fix or mitigation exists

## Please Avoid Posting Publicly

- directly exploitable details
- attack scripts affecting classroom deployments or minors
- specific steps for tampering with builds, packaging, or local configuration
