# 安全政策

如果你发现了可能影响用户、课堂环境或发布产物的安全问题，请不要公开提 issue。

## 私下披露方式

优先顺序：

1. 使用 GitHub 的私密漏洞报告功能（如果仓库已启用）
2. 通过维护者 GitHub 主页提供的联系方式私下联系

## 报告内容建议

- 问题概述
- 影响范围
- 复现步骤
- 受影响平台与版本
- 相关日志、截图或 PoC
- 你认为的风险等级

## 我们的处理目标

- 尽量在 `7` 天内确认收到
- 在复现后评估影响范围与修复优先级
- 在修复或缓解措施准备好后，再决定公开披露节奏

## 不建议公开发布的内容

- 可直接利用的漏洞细节
- 会影响未成年人、学校环境或课堂部署的攻击脚本
- 可用于伪造安装包、篡改更新或窃取配置的具体步骤
