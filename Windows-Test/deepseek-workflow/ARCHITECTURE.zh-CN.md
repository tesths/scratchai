# 教学工作流架构

## 目标

这套架构的核心目标不是“让模型直接产出成品”，而是把 DeepSeek 放在一个可控的教学流程里：

1. 固定题目与课堂约束
2. 先做规划
3. 再拆到角色
4. 再生成学生提示
5. 最后补课堂排错包

这样做的原因是：模型在“完整 Scratch 工程一次性生成”上不稳定，但在“固定题目下的局部教学材料生成”上更可控。

## 分层

### 1. CLI 入口层

- `../run-deepseek-teaching-workflow.mjs`

职责：

- 读取命令行参数
- 创建运行配置
- 调用工作流编排器

这个文件故意保持很薄，避免未来每加一个阶段都改入口逻辑。

### 2. 配置层

- `config.mjs`

职责：

- 解析 `--brief`、`--output-dir`、各阶段模型参数
- 统一生成运行时路径
- 提供默认模型分配

如果后续要支持：

- 多套课堂模板
- 多模型切换
- 远端配置中心

优先从这一层扩展。

### 3. 文件与模板层

- `file-system.mjs`
- `prompts/`

职责：

- 读取/写入 JSON 和文本
- 加载提示词模板
- 管理输出目录与渲染后的提示词副本

为什么把提示词单独拆文件：

- 老师或产品同学可以直接改提示词
- 不需要进主流程代码里找字符串
- 后续可以按“年级 / 学科 / 课程类型”扩展多套模板

### 4. DeepSeek 访问层

- `deepseek-client.mjs`

职责：

- 统一读取 API Key
- 统一调用 `POST /chat/completions`
- 统一处理 JSON 模式与 repair 重试

这里是模型调用的唯一入口。未来如果要：

- 增加日志
- 增加限流
- 增加缓存
- 替换模型供应商

优先改这一层，不要把请求逻辑散到各阶段。

### 5. 校验层

- `core.mjs`

职责：

- 模板渲染
- brief 校验
- 各阶段输出结构校验

注意：

- 当前校验器重点是“结构正确、固定字段不漂移”
- 不是严格语义证明器
- 它的主要作用是给 repair 回路提供确定性的错误清单

未来如果要增强稳定性，可以在这里继续加入：

- 角色级关键字检查
- 广播/变量一致性检查
- 课堂语气检查

### 6. 阶段定义层

- `stages.mjs`

职责：

- 用数据化方式声明每个阶段
- 规定每个阶段用哪套提示词、哪个模型、多少 token、用什么校验器

这是后续扩展的关键层。

如果未来要增加新能力，例如：

- `05-example-block-maps`
- `06-homework-variants`
- `07-assessment-rubric`

理想做法是：

1. 新增提示词模板
2. 在 `stages.mjs` 增加新的 stage builder
3. 在 `workflow.mjs` 编排执行顺序

而不是重写整条流程。

### 7. 工作流编排层

- `workflow.mjs`

职责：

- 串联四个阶段
- 负责阶段间数据传递
- 持久化阶段输出
- 生成最终 summary

当前阶段顺序：

1. `plan`
2. `role-*`
3. `student-hints`
4. `debug-pack`

这个顺序刻意体现“先抽象、再局部、再教学支持”。

### 8. 报告层

- `reporting.mjs`

职责：

- 汇总输出为老师可读的 `SUMMARY.md`
- 生成结果 manifest

把报告逻辑拆出去，是为了以后能同时支持：

- Markdown 总结
- 课堂打印版
- Web 面板展示

## repair 回路

当前模型不稳定点主要有两类：

1. 改题目、改变量、改广播
2. 输出字段漂移

所以架构里默认加入 repair 回路：

1. 先生成
2. 本地校验
3. 如果失败，把错误列表回灌给模型
4. 要求它只修复结构问题

这比单纯堆提示词更稳，也更适合未来继续加校验规则。

## 为什么适合未来扩展

因为现在已经把变化点隔离开了：

- 改模型：改 `config.mjs`
- 改 API：改 `deepseek-client.mjs`
- 改提示词：改 `prompts/`
- 改校验策略：改 `core.mjs`
- 加新阶段：改 `stages.mjs` 和 `workflow.mjs`
- 改输出展示：改 `reporting.mjs`

所以后续如果要从“老师备课工具”扩到：

- 学生分层提示
- 自动生成课堂讲义
- 针对当前 Scratch 项目的实时辅导

都不需要推倒重来。
